1 Executive Summary

Strategic Assessment Overview

This document provides a comprehensive migration analysis for ABB's planned transition from SAP ECC to S/4HANA 2025. The analysis establishes a structured framework for business process mapping, Fit-to-Standard evaluation, Clean Core compliance, and gap remediation — with SAP's Fit-to-Standard methodology and Clean Core principles as the governing priorities.

ABB's ECC system is not connected to C16, which means this analysis operates as a strategic framework and methodology guide rather than a live-data assessment. All process mappings are based on standard SAP ECC-to-S/4HANA change patterns, known custom programs (ZMM_PO_ENHANCE, Z_VENDOR_SCORE, ZMM001, ZMM_VENDOR_REPORT, ZMM_VENDOR_MSME), and SAP's published Simplification List (SAP Note 2313884). Once C16 is connected to ABB's ECC, every assessment in this document can be verified and quantified against live data.

2.2 Migration Objectives (Priority Ordered)

🥇 Priority 1: Fit-to-Standard

Maximize adoption of SAP standard functionality. Retire custom code wherever standard S/4HANA 2025 processes exist. Every custom object must be justified against the standard.

🥈 Priority 2: Clean Core

All retained or new custom code must use only SAP Released APIs (Level A). No direct standard table access, no unreleased APIs, no modifications. Cloud-extensibility ready.

🥉 Priority 3: Process Simplification

Reduce process variants. Harmonize across plants/company codes. Eliminate redundant custom solutions that evolved organically on ECC.

4️⃣ Priority 4: UX Modernization

Migrate from SAP GUI to Fiori Launchpad. Leverage Fiori Elements for new apps. Role-based Fiori spaces for end users.

🚨 Breaking Changes (Will Not Work Without Remediation):
  • Direct SELECT on MSEG — MSEG still exists as a compatibility view but is NOT recommended for new code. MATDOC is the single source of truth.
  • Direct INSERT/UPDATE on KNA1/LFA1 — These tables are read-only in S/4HANA. Must use BP APIs (BAPI_BUPA_CREATE, I_BusinessPartnerTP RAP BO).
  • BSEG for reporting — BSEG exists but must NOT be used for reporting. ACDOCA is the correct source.
  • Classic WM (LE-WM) — Completely removed. No compatibility mode.
  • Costing-based CO-PA — Deprecated. Only account-based CO-PA on ACDOCA.

12 Clean Core Compliance Strategy

SAP's Clean Core framework requires all custom extensions to use only Released APIs — no direct standard table access, no unreleased function modules, no kernel-level APIs, no modifications. This is the foundation for S/4HANA upgrade stability and cloud extensibility.

A Brownfield conversion would carry forward too much technical debt. A pure Greenfield would lose valuable historical data and configuration. Bluefield balances both.
